### 5. 直接结论

#### 5.1 `myl7` 和 `Orca`

在这组公共参数下，`myl7` 在所有已实测点上都明显快于 `Orca`。

最重要的是 `N=10^7`：

- `DPF total`
  - `Orca = 5,280,636 us`
  - `myl7 = 641,560 us`
- `DCF total`
  - `Orca = 5,511,430 us`
  - `myl7 = 555,337 us`

#### 5.2 `libfss`

`libfss` 现在已经功能上接上了这组公共参数：

- `DPF: bin=64`
- `DCF: bin=64, bout=1`

但它是纯 CPU baseline，所以量级明显慢于两套 GPU 实现。

按当前已实测的 `10^5 -> 10^6` 线性关系，`10^7` 的 `total` 预估是：

- `DPF`: `2,235,609,820 us`，约 `37.3 min`，**预估**
- `DCF`: `3,151,336,240 us`，约 `52.5 min`，**预估**

#### 5.3 `Orca / libfss` 优化倍数

这里单独列出来，是因为它正好对应这份横向比较最初的问题之一。

但要强调：

- 这里的 `Orca / libfss` 不是同一个项目内部的 speedup
- 它是**跨实现比较**
- 其中：
  - `libfss` 是 `CPU` 实现
  - `Orca` 是 `GPU` 端到端实现
- 而且下面的表只按当前对齐后的统一参数来算：
  - `DPF: bin=64`
  - `DCF: bin=64, bout=1`
  - `N = 10^5 / 10^6 / 10^7`

计算方式统一为：

- `Orca / libfss 优化倍数 = libfss_total / orca_total`

| primitive | `N=10^5` | `N=10^6` | `N=10^7` |
| --- | ---: | ---: | ---: |
| `DPF` | `22,012,036 / 102,473 = 214.81x` | `223,560,982 / 525,161 = 425.70x` | `2,235,609,820 / 5,280,636 = 423.36x` `预估` |
| `DCF` | `31,723,915 / 66,435 = 477.52x` | `315,133,624 / 565,677 = 557.09x` | `3,151,336,240 / 5,511,430 = 571.78x` `预估` |

这个表最应该怎么读：

- 这是“CPU baseline 换成 GPU 实现后，端到端总时间大约缩了多少倍”
- 所以倍数会非常夸张
- 而且 `DCF` 比 `DPF` 更夸张，因为 `libfss` 的 `DCF` CPU 侧更慢

### 6. 最终结论

- 当前三边已经可以用统一参数做各维度耗时对比。
- 在 `bin=64, bout=1, N=10^5/10^6/10^7` 这组参数下：
  - `myl7` 是更快的 GPU 端到端实现
  - `Orca` 次之
  - `libfss` 可以作为 CPU baseline
- 当前唯一没有继续实跑的是 `libfss 10^7`，本文已经明确标成**预估值**。
